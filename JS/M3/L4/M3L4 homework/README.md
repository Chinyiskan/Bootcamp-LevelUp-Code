### 🐾 Cliente 1 — CliniPet
**Clínica veterinaria · Bogotá, Colombia**

La Dra. Ramírez lleva 8 años atendiendo mascotas con una libreta de apuntes y una hoja de Excel que ya no aguanta más. Necesita un sistema web donde pueda ver todos sus pacientes de un vistazo, filtrarlos, buscarlos por nombre y saber rápidamente cuáles tienen cita próxima o historial crítico.

**Lo que espera ver en pantalla:**
- Tarjetas de cada paciente con foto representativa, nombre, especie, raza, edad y dueño
- Filtros por especie y por estado (activo / dado de alta)
- Buscador por nombre de mascota
- Indicador visual de próxima cita o alerta si tiene historial de urgencias
- Un stat visible con el total de pacientes activos en el sistema

**Una frase suya que debes tener en mente mientras construyes:**
> *"Cuando llega una emergencia no tengo tiempo de buscar. Necesito ver todo de golpe y encontrar al paciente en segundos."*

## El contrato — lo que se evalúa

Independiente del cliente que elijas, tu entrega debe cumplir esto sin excepción:

**Datos bien definidos**
Un array de mínimo 10 objetos con propiedades coherentes y bien nombradas. Sin datos inventados a medias, sin propiedades que no se usan, sin objetos que no tienen la misma estructura.

**Funciones que encapsulan**
Toda la lógica de consulta vive en funciones con `return`. Nada de cálculos sueltos en el orquestador. Si mañana cambia una regla de negocio, solo debes tocar una función.

**El arsenal del módulo aplicado**
Tu proyecto debe usar de forma visible y con sentido: `filter`, `find`, `some`, `sort` y `reduce`. No de relleno — cada uno resolviendo un problema real de tu sistema.

**Comentarios que explican el porqué**
No necesitas comentar cada línea. Sí necesitas explicar las decisiones importantes: por qué ordenas así, qué hace el acumulador, cuándo se activa la alerta.

**Una UI que se ve como producto**
Imágenes reales, paleta de colores coherente con el cliente, tipografía legible, estados vacíos manejados, alertas visibles. Tu instructor va a usar la aplicación — no solo leer el código.

**Repositorio y despliegue**
- Código en GitHub con estructura limpia: `index.html`, `style.css`, `script.js`
- Proyecto desplegado y funcionando en GitHub Pages o Vercel
- README en el repo explicando qué cliente elegiste y qué hace cada función principal
